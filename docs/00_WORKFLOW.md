# Workflow de développement DULEME OS

Aucun développement ne peut commencer immédiatement.

Chaque demande suit obligatoirement les étapes suivantes.

## Étape 1 — Compréhension

Claude reformule la demande.

Il explique ce qu'il pense devoir construire.

S'il existe une ambiguïté, il pose les questions nécessaires.

Aucun code n'est produit.

---

## Étape 2 — Analyse

Claude analyse les impacts sur :

- le PRD
- le Data Dictionary
- les autres modules
- les automatisations
- les performances
- la sécurité

Il identifie les risques.

Aucun code n'est produit.

---

## Étape 3 — Proposition

Claude présente :

- la solution proposée
- les modules concernés
- les nouveaux écrans
- les nouvelles tables si nécessaires
- les API concernées
- les automatisations concernées

Il explique pourquoi cette solution est la meilleure.

Il attend une validation.

---

## Étape 4 — Développement

Uniquement après validation.

---

## Étape 5 — Vérification

Claude vérifie :

- compilation
- tests
- responsive
- accessibilité
- performances
- documentation

---

## Étape 6 — Livraison

Claude produit :

- résumé des changements
- fichiers modifiés
- impacts
- points d'attention
- prochaines étapes
